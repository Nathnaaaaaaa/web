Client server

↑click pic for more info↑

Client: This is the device or software that makes requests. For example, your web
browser is a client when you visit a website. It asks the server for the
information (like a webpage).

Request and Response: The client sends a request (e.g., "Show me this webpage"), and
the server responds with the necessary data (e.g., the content of the
webpage).

Examples: Web browsers (client) communicate with web servers to fetch
webpages. Email clients (like Outlook or Gmail) communicate with email
servers to send/receive messages.





Mesh Network

↑click pic for more info↑

Mesh Networks: A mesh network is a type of network topology where each device (node) is
connected to multiple other devices, creating a web-like structure. This ensures better
coverage and redundancy.

Nodes: In a mesh network, each device or "node" acts as both a client and a relay point.
This means that each device can send and receive data, and can also forward data from other
devices in the network.





Web server

↑click pic for more info↑

A web server delivers web pages to clients (browsers) over the internet. Clients send HTTP requests
to the server, which processes them and responds with the requested content. The browser then displays
the content to the user.
